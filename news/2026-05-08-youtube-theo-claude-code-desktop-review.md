Theo Browne (t3.gg), creator of T3 Code and one of the most technically credible reviewers in the TypeScript/React space, published his first hands-on review of the Claude Code desktop app that shipped at Anthropic's Code with Claude conference. His verdict: it remains the most capable AI coding environment he uses daily, but the desktop app launch was plagued by UX bugs, missing basic features, and patterns that feel untested — a surprising quality gap from a company that just hosted a developer conference.

The video is notable because Theo does not do hype. He identifies specific broken interactions and compares them to what T3 Code and Cursor handle better, making this a useful reference for developers evaluating whether to adopt the native desktop client versus continuing with the CLI or IDE extension. He also briefly discusses the broader question of whether Anthropic is spreading too thin across product surfaces.

For Claude Code users deciding whether to switch from the VS Code extension or CLI to the desktop app, this review is the most technically grounded take available within 48 hours of launch.
